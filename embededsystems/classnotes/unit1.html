an embeded system is an electronic system designed to perform a specific function with a combination of both hardware and software.

Embedded system. General computer systems.
A system which is a combination of special purpose hardware and embedded operating system for executing a specific set of applications. Combination of generic hardware and a general purpose operating system for executing a variety of applications.
May or may not contain an operating system. Contains a general purpose operating system.
Operating system is pre programmed and is non alterable by end user. There may be exceptions such as kernel flashing through system hardware settings. Applications are iterable- programmable by end user.
Application specific requirements are key deciding factor. Performance is key factor in selection of system.
Highly tailored to take advantage of the power saving modes supported by hardware and operating system. Less or not at all tailored to reduce power requirement options for different levels of power management.
For certain category of embedded systems like mission critical systems, the response time requirement is highly critical. Response requirements are not time critical.
Execution Behaviour's is deterministic for certain type of embedded systems like hard real time system. Need not be deterministic and execution behaviour.

Microprocessor v/s Microcontroller

Microprocessor microcontroller
A silicon chip representing A central processing unit which is capable of performing arithmetic as well as logical operations according to predefined set of instructions. A microcontroller is a highly integrated chip that contains a CPU, scratchpad RAM, special and general purpose register arrays, on CHIP ROM/flash memory for programme storage, timer and interrupt control units and dedicated IO ports.
It is a dependant unit.It requires combination of other chips like timer, programme and data memory chips, interrupt controllers, etc for functioning. It is a self content unit and it doesn't require external interrupt controller, timer, UART (universal asynchronous receiver and transmitter), etc for its functioning.
Most of the time general purpose in design and operation. Mostly application oriented or domain specific.
Doesn't contain built in IO port. The IO port functionality needs to be implemented with the help of external programmable peripheral interface chips like 8255. Most of the processes contains multiple built in IO ports which can be operated as a single 8/16/32 bit port or as individual port pins.
Targeted for high end markets where performance is important. Targeted for embedded markets where performance is not so critical.
Limited power saving options compared to microcontrollers. Includes a lot of power saving features.
